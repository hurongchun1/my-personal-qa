# RAG 深入研究计划 - 第二阶段

## 一、研究目标

基于第一阶段的基础实现，深入研究RAG技术栈的各个环节，提升知识库问答系统的质量和效果。

## 二、当前实现分析

### 2.1 已实现功能
- **文档解析**：PDF、HTML、Markdown三种格式
- **文本切分**：字符分割、语义分割、Token分割三种策略
- **Embedding模型**：DashScope `text-embedding-v1`
- **向量存储**：FAISS、Elasticsearch
- **检索生成**：简单相似性搜索 + DashScope LLM

### 2.2 主要不足
1. **Embedding模型单一**：仅支持DashScope，缺乏本地模型支持
2. **缺少Rerank机制**：检索后直接使用top-k结果，没有重排序
3. **切分策略有限**：缺乏深度文档理解（版面分析、OCR等）
4. **切分效果无评估**：没有测试切分质量的方法
5. **向量数据库选择有限**：仅FAISS和ES，缺乏专业向量数据库支持

## 三、深入研究方向

### 3.1 Embedding模型研究

#### 当前状态
- 使用DashScope `text-embedding-v1`模型
- 维度：1536维
- 支持中英文

#### 研究方向
1. **本地模型部署**
   - BGE系列：`bge-large-zh-v1.5`（中文优化）、`bge-m3`（多语言+长文本）
   - M3E系列：`m3e-large`（中文语义匹配）
   - text2vec系列：`text2vec-large-chinese`

2. **模型对比测试**
   - 在相同数据集上测试不同模型的检索效果
   - 评估维度：准确率、召回率、F1值
   - 考虑因素：模型大小、推理速度、内存占用

3. **混合Embedding策略**
   - 结合稠密向量和稀疏向量（BM25）
   - 多向量检索（ColBERT风格）

#### 实施步骤
```
1. 调研主流开源Embedding模型
2. 搭建本地模型部署环境
3. 设计评估数据集和指标
4. 对比测试不同模型效果
5. 选择最优模型集成到系统
```

### 3.2 Rerank模型研究

#### 当前状态
- 无Rerank机制
- 检索后直接使用top-k结果

#### 研究方向
1. **开源Rerank模型**
   - `bge-reranker-v2-m3`：中文轻量化部署首选
   - `Qwen3-Reranker-4B`：中文高精度首选
   - `jina-reranker-v3`：智能代理场景优化
   - `ms-marco-MiniLM-L6-v2`：英文轻量级

2. **混合检索策略**
   - 向量检索 + 关键词检索
   - 混合评分公式：`混合分 = (文本相似分 × 文本权重) + (向量相似分 × 向量权重)`
   - 默认权重：文本权重0.3，向量权重0.7

3. **重排序流程**
   ```
   初步召回 → 混合评分 → 过滤低分 → 重新排序 → Top-K结果
   ```

#### 实施步骤
```
1. 调研主流Rerank模型
2. 实现混合检索机制
3. 集成Rerank模型
4. 设计评估指标（MRR、NDCG）
5. 优化权重参数
```

### 3.3 文档切分策略研究

#### 当前状态
- 字符分割：RecursiveCharacterTextSplitter
- 语义分割：SemanticChunker
- Token分割：TokenTextSplitter

#### 研究方向
1. **深度文档理解**
   - 版面分析：识别标题、段落、表格、图片等区域
   - OCR识别：提取扫描件或图片中的文字
   - 表格结构识别：解析表格的行列结构

2. **智能切分策略**
   - 按段落分割：保留自然段落结构
   - 按句子分割：保持句子完整性
   - 按标题层级分割：适用于Markdown、HTML
   - 表格专项处理：表格内容单独处理

3. **切分效果测试**
   - **结构指标**：块大小分布、大小合规率、重叠准确率、空/微块率、边界完整性
   - **内容指标**：语义连贯性、上下文自足性、信息完整性、边界质量、跨块冗余度
   - **质量等级**：A（优秀）≥90、B（良好）80-89、C（合格）70-79、D（较差）60-69、F（不合格）<60

#### 实施步骤
```
1. 研究DeepDoc等深度文档理解技术
2. 实现更多切分策略（段落、句子、标题等）
3. 设计切分效果评估体系
4. 开发切分质量测试工具
5. 优化切分参数和策略
```

### 3.4 向量数据库研究

#### 当前状态
- FAISS：本地文件存储
- Elasticsearch：分布式搜索引擎

#### 研究方向
1. **专业向量数据库**
   - Milvus：云原生分布式向量数据库
   - Chroma：轻量级嵌入式向量数据库
   - Qdrant：高性能向量相似性搜索引擎

2. **混合查询能力**
   - 向量 + 标量过滤
   - 全文检索 + 向量检索
   - 多条件组合查询

3. **性能优化**
   - 索引类型选择：IVF、HNSW、PQ等
   - GPU加速支持
   - 分布式扩展能力

#### 实施步骤
```
1. 调研主流向量数据库
2. 搭建测试环境
3. 设计性能测试用例
4. 对比测试不同数据库
5. 选择最优方案集成
```

## 七、风险与应对

### 7.1 技术风险
- **模型部署复杂**：提供Docker镜像和详细部署文档
- **性能瓶颈**：采用异步处理和缓存机制
- **兼容性问题**：设计良好的接口抽象和适配器模式

### 7.2 时间风险
- **任务延期**：设置合理的里程碑和缓冲时间
- **优先级调整**：根据实际进展动态调整计划

### 7.3 资源风险
- **计算资源不足**：采用云服务或优化模型大小
- **数据集缺乏**：使用公开数据集和合成数据
