# RAG 技术分析总结

## 一、当前项目状态

### 1.1 已实现功能
- **文档解析**：支持PDF、HTML、Markdown三种格式
- **文本切分**：字符分割、语义分割、Token分割三种策略
- **Embedding模型**：DashScope `text-embedding-v1`（1536维）
- **向量存储**：FAISS（本地文件）、Elasticsearch（分布式）
- **检索生成**：简单相似性搜索 + DashScope LLM

### 1.2 主要优势
1. **架构清晰**：采用策略模式和工厂模式，易于扩展
2. **接口统一**：BaseLoader抽象基类，统一parse()接口
3. **配置灵活**：支持多种向量存储和切分策略

### 1.3 主要不足
1. **Embedding模型单一**：仅支持DashScope，缺乏本地模型支持
2. **缺少Rerank机制**：检索后直接使用top-k结果，没有重排序
3. **切分策略有限**：缺乏深度文档理解（版面分析、OCR等）
4. **切分效果无评估**：没有测试切分质量的方法
5. **向量数据库选择有限**：仅FAISS和ES，缺乏专业向量数据库支持

## 二、RAGFlow源码研究

### 2.1 核心设计理念
RAGFlow的核心设计理念是 **"Quality in, quality out"**，其架构最大特点是**重度投入在文档的细粒度解析上**，而非单纯依赖通用的RAG组件。

### 2.2 技术栈与模块
- **Web服务**：Flask
- **业务数据库**：MySQL
- **向量数据库**：ElasticSearch
- **文件存储**：MinIO
- **核心模块**：
  - `api`：后端API
  - `deepdoc`：文件解析模块（核心亮点）
  - `rag`：核心检索与生成逻辑
  - `graphrag`：知识图谱实现

### 2.3 关键技术特点
1. **无RAG中间件**：没有使用LangChain或LlamaIndex，完全自研
2. **异步任务处理**：文档解析是计算密集型操作，采用异步任务队列
3. **混合检索**：完全依赖ElasticSearch实现文本与向量的混合检索
4. **深度文档理解**：通过DeepDoc实现OCR、版面分析、表格识别

### 2.4 文档切分流程
1. **触发与分片**：上传文件后手动触发解析，根据文件类型和解析器类型拆分为多个异步任务
2. **核心解析能力**：
   - 文档版面分析（Layout Analysis）
   - OCR识别
   - 内容提取与清洗
   - 结构关联（独特亮点：文本块与原始文档物理位置关联）

### 2.5 Rerank机制
RAGFlow的重排是一个**混合评分、过滤和再排序**的过程：
1. **评分公式**：`混合分 = (文本相似分 × 文本权重) + (向量相似分 × 向量权重)`
2. **默认权重**：文本权重0.3，向量权重0.7
3. **过滤机制**：默认过滤掉混合得分低于0.2的文档

## 三、深入研究方向

### 3.1 Embedding模型研究

#### 当前状态
- 使用DashScope `text-embedding-v1`模型
- 维度：1536维
- 支持中英文

#### 研究方向
1. **本地模型部署**
   - BGE系列：`bge-large-zh-v1.5`（中文优化）、`bge-m3`（多语言+长文本）
   - M3E系列：`m3e-large`（中文语义匹配）
   - text2vec系列：`text2vec-large-chinese`

2. **模型对比测试**
   - 在相同数据集上测试不同模型的检索效果
   - 评估维度：准确率、召回率、F1值
   - 考虑因素：模型大小、推理速度、内存占用

3. **混合Embedding策略**
   - 结合稠密向量和稀疏向量（BM25）
   - 多向量检索（ColBERT风格）

### 3.2 Rerank模型研究

#### 当前状态
- 无Rerank机制
- 检索后直接使用top-k结果

#### 研究方向
1. **开源Rerank模型**
   - `bge-reranker-v2-m3`：中文轻量化部署首选
   - `Qwen3-Reranker-4B`：中文高精度首选
   - `jina-reranker-v3`：智能代理场景优化
   - `ms-marco-MiniLM-L6-v2`：英文轻量级

2. **混合检索策略**
   - 向量检索 + 关键词检索
   - 混合评分公式：`混合分 = (文本相似分 × 文本权重) + (向量相似分 × 向量权重)`
   - 默认权重：文本权重0.3，向量权重0.7

3. **重排序流程**
   ```
   初步召回 → 混合评分 → 过滤低分 → 重新排序 → Top-K结果
   ```

### 3.3 文档切分策略研究

#### 当前状态
- 字符分割：RecursiveCharacterTextSplitter
- 语义分割：SemanticChunker
- Token分割：TokenTextSplitter

#### 研究方向
1. **深度文档理解**
   - 版面分析：识别标题、段落、表格、图片等区域
   - OCR识别：提取扫描件或图片中的文字
   - 表格结构识别：解析表格的行列结构

2. **智能切分策略**
   - 按段落分割：保留自然段落结构
   - 按句子分割：保持句子完整性
   - 按标题层级分割：适用于Markdown、HTML
   - 表格专项处理：表格内容单独处理

3. **切分效果测试**
   - **结构指标**：块大小分布、大小合规率、重叠准确率、空/微块率、边界完整性
   - **内容指标**：语义连贯性、上下文自足性、信息完整性、边界质量、跨块冗余度
   - **质量等级**：A（优秀）≥90、B（良好）80-89、C（合格）70-79、D（较差）60-69、F（不合格）<60

### 3.4 向量数据库研究

#### 当前状态
- FAISS：本地文件存储
- Elasticsearch：分布式搜索引擎

#### 研究方向
1. **专业向量数据库**
   - Milvus：云原生分布式向量数据库
   - Chroma：轻量级嵌入式向量数据库
   - Qdrant：高性能向量相似性搜索引擎

2. **混合查询能力**
   - 向量 + 标量过滤
   - 全文检索 + 向量检索
   - 多条件组合查询

3. **性能优化**
   - 索引类型选择：IVF、HNSW、PQ等
   - GPU加速支持
   - 分布式扩展能力
